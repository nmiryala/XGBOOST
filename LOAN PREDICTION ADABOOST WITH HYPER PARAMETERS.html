LOAN PREDICTION USING ADABOOST¶

Boosting is another type of ensemble technique which combines multiple weak classifiers (which is slightly better than a random guessing) into a single strong classifier. We can use any other classification algorithm (Logistic Regression or Decision Tre) as a base for boosing and it is called as base classifier.

The general idea of the Boosting algorithms is to build multiple classifiers in a sequential manner where each subsequent model attempts to fix the errors of its predecessors. Boosting as opposed to bagging, in which classifiers are built in parallel.

Boosting builds initial classifier by giving equal weights to each sample and then focuses on correctly classifying misclassified examples in subsequent classifiers.

Adaptive Boosting which is most commonly called as AdaBoost, is a Boosting Algorithm. The method this algorithm uses to correct its predecessor is by paying more attention to underfitted training instances by the previous model.

At the end of every model prediction we end up boosting the weights of the misclassified instances so that the next model does a better job on them and this process continues.

Gradient Descent vs AdaBoost:

GD tweaks a single predictor's parameter to minimize the cost function where as AdaBoost adds predictors to the ensemble gradually making it better.

Great disadvantage with the model is that the model cannot be parallelized since each predictor can only trained after the previous one has been trained and evaluated.

Steps to be used for performing the AdaBoost Algorithm:¶

  1. Initially all observations are given equal weights

  2. A model is built on a subset of data

  3. Using this model, predictions are made on the whole dataset

  4. Errors are calculated by comparing the predictions and actual values

  5. While creating the next model, higher weights are given to the data points which were predicted incorrectly.

  6. Weights can be determined using the error value. For instance, the higher the error the more is the weight assigned to the observation.

  7. This process is repeated until the error function does not change, or the maximum limit of the number of estimators is reached.

Hyperparameters Used¶

  1. base_estimators:Object- specifies the base type of estimator, i.e. the algorithm to be used as a base learner. This is the base estimator from which the boosted ensemble is built. The estimator should have the capability to utilize differential weights for samples (for example, logistic regression or decision tree)
  1. n_estimators:Integer- It defines the number of base estimators, where the default is 10. But we can increase it in order to attainbetter performance. This is the maximum number of estimators at which boosting algorithm is terminated. In the case of perfect fit, the learning procedure is stopped early.
  1. learning_rate:Float- It controls the contribution of weak learners in the final combination. Learning Rate shrinks the contribution of each classifier specified by learning_rate. There is a trade-off between learning_rate and n_estimators. A paper by Friedman (1999), suggest to set a large value for the number of trees then tune the shrinkage parameter to achieve the best results. Studies in the paper preferred a shrinkage value of 0.1, a number of trees in the range 100 to 500 and the number of terminal nodes in a tree between 2 and 8.
  1. max_depth:int or None, optional (default=None) The maximum depth of the individual estimator (Decision Tree). If None, then nodes are expanded until all the leaves contain less than min_samples_split samples. The higher value of maximum depth causes overfitting, and a lower value causes underfitting
  1. random_state : makes the model’s output replicable. It will always produce the same results when you give it a fixed value as well as the same parameters and training data.
  1. n_jobs : indicates the system how many processors it is allowed to use. Value of ‘-1’ means there is no limit;

## Importing the metrics
from sklearn import metrics
## Defining the matrix to draw the confusion matrix from actual and
## predicted class labels
def draw_cm( actual, predicted ):
# Invoking confusion_matrix from metric package. The matrix
# will be oriented as [1,0] i.e. the classes with label 1 will be
# represented by the first row and 0 as second row
    cm = metrics.confusion_matrix( actual, predicted, [1,0] )
# Confusion will be plotted as heatmap for better visualization
# The labels are configured to better interpretation from the plot
    sn.heatmap( cm, annot=True, fmt='.2f',
    xticklabels = ["Approved", "Not Approved"],
    yticklabels = ["Approved", "Not Approved"] )
    plt.ylabel("True label")
    plt.xlabel("Predicted label")
    plt.show()
